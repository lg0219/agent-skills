# Data Table Query 技能

统一处理钉钉 AI表格、在线表格(axls)、本机 Excel 的数据**读写操作**，保证字段映射准确、空值不猜测、写入不破坏。

## 适用范围

| 数据来源 | 示例 | 读 | 写 |
|---------|------|----|----|
| 钉钉 AI表格 | ait table `资产表` | `dws aitable record query` | `dws aitable record create/update/delete` |
| 钉钉在线表格 | axls `销售数据` | `dws sheet read` | `dws sheet write / rows-insert / rows-delete` |
| 本机 Excel 文件 | `data/销售.xlsx` | `pandas.read_excel()` | `df.to_excel()` |

## 标准流程

前置核心：**无论读写，先拿到字段映射。**

### 第一步：定位数据源

**AI表格 →** 搜关键词找 base → `aitable base get` 拿 tableId + 字段结构
**在线表格 →** 找文件路径/ID → `dws sheet info` 拿 sheetId + 表头
**本机Excel →** 找文件路径 → pandas 读列名

### 第二步：存/取字段映射

路径：`data-tables/{来源}_{表名}.json`

```json
{
  "source": "aitable|sheet|excel",
  "baseName": "IT资产管理",
  "tableName": "资产表",
  "lastUpdated": "2026-06-03",
  "fieldCount": 13,
  "recordCount": 12,
  "fields": {
    "原始字段ID": { "name": "显示名称", "type": "text|number|date|singleSelect|user|currency" }
  }
}
```

**规则：**
- 已存过的表直接读映射，不重新查
- 需要最新结构时强制刷新（用户说了算）
- 写入后更新 `recordCount`

---

## 读取操作

### 第三步：读取数据

用对应工具拉取数据，最大 100 条/次，超限分页。

### 第四步：逐行验证

每条记录逐字段检查：

```
for each record:
  for each field:
    字段不存在 → 留空（不猜、不补）
    有值       → 取实际值
    空字符串   → 留空
```

### 第五步：展示

- ≤15 条：完整表格
- >15 条：汇总统计 + 抽样展示
- 需出图/报告：按需处理

---

## 写入操作

### 新增记录

**前置确认（必须做）：**
1. 向用户展示要填入的字段和值
2. 确认哪些字段必填、哪些可空
3. 用户确认后才执行

**执行后展示：**
- 展示实际写入的结果（字段名+值）
- 确认写入成功

**字段值格式注意事项：**

| 类型 | 写入格式 | 示例 |
|------|---------|------|
| text | 字符串 | `"钉钉专业版"` |
| number | 数字 | `100` |
| currency | 数字（注意单位） | `50000` |
| date | ISO 日期字符串 | `"2026-06-03T00:00:00+08:00"` |
| singleSelect | 选项名（需确认选项存在） | `"购买"` |
| user | userId 或邮箱 | `"0205446618840844"` |

### 修改记录

**前置确认：**
1. 先读当前数据展示给用户（**以表格形式展示所有字段，新旧值对照**）
2. 确认要修改哪条记录的哪些字段
3. 用户确认后才执行

**执行后展示：**
- **展示该记录的所有字段**（不只是改动的字段），以表格形式列出全部字段名和值
- 确认修改成功
- 使提出者能清楚看到修改后的完整数据

**流程：**
```
定位记录(recordId) → 读取完整数据 → 表格展示(全部字段) → 用户确认 → 执行update → 读回验证 → 表格展示修改结果(全部字段)
```

### 删除记录

**⚠️ 危险操作，必须：**
1. 展示要删除的记录完整内容
2. 用户明确回复"确认"/"删除"
3. 加 `--yes` 执行
4. 删完后验证剩余数据

**执行后展示：**
- 确认删除成功
- 展示删除后的剩余记录数
- 如有影响，说明影响范围

### 写入后验证

每次写入操作后，必须重新读取该条数据验证写入结果，确认：
- 值已正确写入
- 未影响到其他数据
- 格式符合预期

## 写入字段确认规则

**提出者请求中包含了不存在的字段或你无法确定的字段时：**

1. 先查当前表的字段映射
2. 对比请求中有哪些字段是表中**没有的**或者**不确定对应关系**的
3. 向提出者展示这些存疑字段，询问确认后再决定是否写入

> 示例：请求说要填"所属部门"，但表中没有这个字段 → 问提出者是不是指现有的"存放位置"，还是需要新增字段

## 模糊指令处理规则（真实场景）

用户可能不是 hunter，而是普通员工。他们的指令可能不准确、带有口音或错别字。

| 问题 | 应对方式 |
|------|---------|
| 表名说错或模糊 | 展示匹配到的表名列表，让用户选择 |
| 字段名不准确/错别字 | 复述你理解的字段名给用户确认，如"你是想改'服务器名称'吗？" |
| 操作意图不清 | 先读当前数据展示给用户，让ta确认要改哪条、改什么 |
| 指令过于简短 | 反问引导："你要查数据、新增、修改还是删除？" |
| 指令矛盾或超出权限 | 说明限制并给出可行建议，不自行推断执行 |

**核心原则：任何写操作，必须先读当前数据展示给用户，ta确认后再执行。不能相信任何未经确认的指令。**

## 注意事项

1. **AI表格的 user 类型字段**：写入时用 userId，展示时可根据需要查姓名
2. **日期格式**：统一用 ISO 格式 `2026-04-28T00:00:00+08:00`
3. **金额单位**：首次接触时先读一条确认数字单位（元/分）
4. **分页**：单次最多 100 条，超限游标翻页
5. **singleSelect 字段**：写入前确认选项存在，不存在需要先加选项
6. **覆盖保护**：修改前必须展示当前值 + 新值对比，用户确认再执行
7. **映射更新**：如果表结构变更（增删字段），刷新映射文件，不是覆盖
